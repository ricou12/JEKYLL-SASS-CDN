Liens utiles

Jekyll - Transform your plain text into static websites and blogs.

Introduction à Liquid

  • Accélère la productivité de projet web
  • Converti les fichiers markdown en html (langage de balisage plus léger que html)
  • Embarque Sass (préprocesseur CSS)
  • Intègre le moteur de template Liquid

Démarrage

  • Avec le terminal d'un éditeur de code
  • Avec une console (Bash, Shell ...)

Installation

Jekyll est écrit en Ruby : Installer windows

gem install bundler jekyll : Installe Jekyll

jekyll new mon-site-web : Pour créer un nouveau projet

cd mon-site-web : Pour se placer dans le repertoire du projet.

bundle exec jekyll serve : Install et execute le server.

http://localhost:4000 : Pour consulter ses pages.

Fonctionalités

Les dossiers du projet

sans underscores "_" seront générer en l'état dans le dossier site, avec "_" servent lors de la compilation.

  • assets dans la racine du projet, créer des sous-dossiers assets pour stocker vos dépendances les fichiers CSS, JS, vos images...
  • _includes : morceaux de codes liés à une page html
  • _layouts : pages md ou html principale et autre bloc de code integrant les includes et traitement de données avec Liquid.
  • _sass : Regroupe les fichiers *.scss (feuille de style)
  • _post : pour stocker des articles type Markdown (*.md) dans le cadre de blog
  • _data : pour stocker des tableaux de données dans des fichiers type yaml, gson ...

SASS

Jekyll embarque SASS.
Jekyll génére automatiquement les fichiers css à partir de vos fichiers scss.

Vou pouvez éditer plusieurs fichiers scss dans le dossier "_sass" pour alléger le code, le rendre plus clair, accessible à la lecture et séparer les fonctionalités.
Pour cela éditez un fichier scss dans le dossier assets par exemple
Importer toutes les dépendances que vous avez edité dans le dossier _sass.

Préférer par usage de créér un fichier principal scss dans le dossier assets (ex: assets/css/main.scss) dans lequel figure tout les imports *.scss lié à votre projet. Ce fichier devra contenir un Front Matter en-tete necessaire pour la génération du fichier css final.

               ---
               # front matter
               ---
               // assets/css/main.scss
               @import "./style1.scss";
               @import "./style2.scss";
               etc ...
