SASS

Syntactically Awesome Style Sheets

Définition

C'est un pré-processeur pour le langage CSS.
Il permet de générer dynamiquement du code CSS tout en offrant une syntaxe simple et un code facilement réutilisable et maintenable.

Installation locale, inclus les dépendances du projet (node_module)

  • Installer Ruby en le téléchargeant à partir du lien suivant https://rubyinstaller.org/
  • Créer un dossier pour le projet, puis l'ouvrir dans l'éditeur.
  • Initialiser le projet, créer un fichier "package.json". 
    npm init -y
    L'option "-y" permet de reporter la phase de configuration du fichier package.json.
    • Il regroupe des informations sur:
      • Le nom du projet.
      • La version.
      • Description du projet.
      • L'auteur.
    • Permet de lancer des scripts.

      Ex: automatiser la compilation, relancer le serveur...

    • Conserve les dépendances du projet .

      Par la suite si l'on souhaite effectuer un clonage du projet à partir de Github on utilisera la commande "npm install" pour installer automatiquement les dépendances vers le dossier node_module.

  • Installer node-sass pour compiler les fichiers scss au format css. En effet les navigateurs sont incapables d'interprêter ce type fichier. 
    npm install node-sass --save-dev

    L'option: "--save-dev" ou plus simplement "-D" ajoute une dépendance au fichier "package.json" dite dev-time, exécutée pendant la phase de développement.

    L'option: "--save" ou plus simplement "-s" crée une dépendance au fichier "package.json" dite runtime (en temps réelle) ex: bootstrap, jquery ou tout autre dépendance necessaire au bon fonctionnnemnt des pages web.

  • Automatiser la compilation. Ajouter à la section script du fichier package.json 
    "compile-sass": "node-sass sass/main.scss css/main.css --watch"
  • Définir le dossier cible pour la génération du css, ne pas oublier d'ajouter le lien du css dans la balise < head >. 
    <link rel="stylesheet" type="text/css" href="css/main.css"/>
  • Démarrer la compilation en mode auto si renseigné dans le "package.json" (générer le fichier css à partir du scss). 
    npm run compile-sass

  • Intégrer une version de Bootstrap avec ses dépendances dans son projet.
    • Avec le CDN
        • bootstrap.min.css (Dans la balise < head >).
        • bootrap.min.js (dans la balise < script > avant < body >)
        • popper.js (idem)
        • jquery.js (idem)
    • Importer à partir des dépendances de node_modules, les fichiers scss de Bootstrap.
      Ouvrir node_modules/bootstrap/scss le fichier bootstrap.scss qui s'y trouve doit être importé dans notre fichier main.scss en premier ligne afin que les règles de bootstrap prime sur nos propes règles.

    Les fonctionalités (règles)

    • @use charge les mixins, les fonctions et les variables d'autres feuilles de style Sass et combine les feuilles CSS de plusieurs feuilles de style.
    • @forward charge une feuille de style Sass et rend ses combinaisons, fonctions et variables disponibles lorsque votre feuille de style est chargée avec la règle @use .
    • @import étend la règle CSS at pour charger des styles, des mixins, des fonctions et des variables à partir d'autres feuilles de style.

      • Ici l'ensemble des fichiers scss sont importés vers le fichier principal. Lors de la compilation l'intégralité des fichiers scss importés seront ajouté au fichier css final.

    • @mixin et @include facilitent la réutilisation de morceaux de styles.
    • @function définit les fonctions personnalisées pouvant être utilisées dans les expressions SassScript .
    • @extend permet aux sélecteurs d'hériter des styles les uns des autres.
    • @at-root place les styles en son sein à la racine du document CSS .
    • @error provoque l'échec de la compilation avec un message d'erreur.
    • @warn un avertissement sans arrêter complètement la compilation.
    • @debug imprime un message à des fins de débogage.
    • Des règles de contrôle de flux telles que @if , @each , @for et @while contrôlent le nombre de fois ou plus de styles émis.
